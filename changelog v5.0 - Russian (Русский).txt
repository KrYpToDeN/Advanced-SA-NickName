Перевод плагина версий 1.0 - 4.0 реализован только на русском языке!
В данных версиях реализована поддержка никнеймов только с русскими, английскими и стандартными символами в SA:MP!
Начиная с версии 4.0, благодаря Regex, реализована поддержка любых символов в никнеймах!
Начиная с версии 5.0 перевод плагина реализован на русском и английском языках!


Версия 1.0
	1. Создан плагин Vodka_SAMP
	2. Автоматический поиск адресов памяти. Работает абсолютно на любой версии сервера
	3. Распознование версии сервера(только для Windows)
	2. Исходный код закрыт.

Версия 2.0
	1. Добавлена возожность использовать русские буквы ё-Ё
	2. Устранён баг с длиной ника. Не забываем вписывать инклуд во всех Filterscripts и GameMode.
	
Версия 3.0
	1. Отрытие исходного кода. Размещение на GitHub
	2. Обновлён проект под версию Visual Studio 2015
	3. Добавлена возможность управлять плагином в конфигурационном файле(scriptfiles/Vodka_SAMP.ini)
	4. Частично переписан код
	5. Добавлена возможность автоматического определения файла запуска сервера
	6. Исправлен баг, где версия сервера не показывалась в клиенте 0.3.7
	(проверка версии полностью переписана)
	7. Добавлен показ версии сервера для ОС Linux
	8. Полностью переписана функция проверки ника
	9. Улучшен хук для сервера
	10. Исправлена ошибка хука в инклуде для сервера
	11. В инклуд для сервера добавлена функция 'IsValidNickName(name[])' для проверки валидности ника
			Возвращает: true - ник валиден, false - в нике запрещённые символы
	12. Оптимизирован и улучшен код.

Версия 4.0
	1. Переименование плагина и всех его составляющих (инклуд, файл настроек) с Vodka_SAMP на Advanced-SA-NickName (сокращённо ASAN)
	2. Добавлена поддержка стандартов C++11 в makefile
	3. Обновлены инклуды для работы с INI. Код переписан под язык C++ (был под C)
	4. Теперь ник проверяется на валидность с помощью Регулярных выражений (Regex) https://ru.wikipedia.org/wiki/Регулярные_выражения
	5. Проверка ника на валидность с помощью функции 'IsValidNickName(name[])' теперь тоже работает с помощью Regex.
	6. Функция проверки ника 'IsValidNickName(name[])' перенесена в плагин. В инклуде для сервера вынесен native
	7. Исправлен баг в хуке функции 'GetPlayerName(..)' в инклуде. Баг был в том, что функция не возвращала длину ника.
	8. Изменён файл настроек плагина. В него включены следующие настройки:
		
		EnginePlugin
		- Включить плагин - 1, Отключить плагин - 0
		
		RegexTemplate
		- Регулярное выражение (Включая регистр символов)
		
		AllowSpace
		- Заменить символ нижнего подчёркивания(_) на символ пробела( ) - 1, По умолчанию - 0
		- Пример. Если 'AllowSpace = 1' -> 'John_Connor' будет в табе как 'John Connor' после коннекта. НЕ ИСПОЛЬЗУЙТЕ ПРОБЕЛЫ В ОКНЕ САМПА! ТОЛЬКО СЕРВЕРНАЯ СТОРОНА!
		
		MaxSpaces
		- РАБОТАЕТ ТОЛЬКО, ЕСЛИ `AllowSpace` = 1.
		- Количество замен символа (_) на символ (). Тоесть если стоит 1, то система заменит символ только 1 раз, а остальные пропустит!
	
	9. Некоторые изменения в коде, что добавляет удобства для чтения кода другим программистам.
	
	Для обновления плагина со старых версий, обязательно:
		1. Удалите файл настроек 'Vodka_SAMP.ini', плагин 'Vodka_SAMP.dll' (.so на Linux), инклуд 'Vodka_SAMP.inc'
		2. Положите новый плагин, инклуд в нужные папки.
		3. В Ваших GameMode/FilterScripts измените 'include "Vodka_SAMP"' на 'include "ASAN"'
		4. В сервер.cfg, в строке 'plugins', измените 'Vodka_SAMP' на 'ASAN'
		
Версия 5.0 - Самое масштабное и, наверное, последнее обновление плагина.
	1. Добавлена возможность входа под тем же ником, который присутствует на сервере, с возможностью контролирования кол-во одновременных ников, а также регистра символов у повторного ника;
	2. Установка длины ника больше не работает через Regex. Для этого используйте файл конфигураций;
	3. Добавлена возможность изменения длины ника в SA:MP. Теперь Вы можете установить в файле конфигураций минимальную и максимальную длину ника (от 1 до 24 символов);
	4. Добавлена возможность замены ВСЕХ символов нижнего подчеркивания на символы пробела. В прошлой версии присутствовало ограничение по количеству замен. Настройка производится в файле конфигураций;
	5. Добавлен англ.язык в логах плагина. В файле конфигураций присутствует выбор требуемого языка;
	6. Инклуд для плагина теперь разбит на 2 части. 1 - ASAN.inc для GameMode и ASAN_FS.inc для FilterScripts;
	7. Добавлены хуки для OnPlayerConnect, OnPlayerDisconnect, SetPlayerName;
	8. Обновлен инклуд файл, добавлено 2 хука для работы с пунктом 1;
	9. native функция IsValidNickName(name[]) заменена на ASAN_IsValidNickName(name[]) из за несовместимости с плагином YSF;
		Если вы используете эту функцию из прошлой версии плагина, переименуйте ее в Ваших GameMode/FilterScipts; 
	10. Настройки 'AllowSpace' и 'MaxSpaces' из файла конфигураций заменены на 'MaxAllowdedSpaces';
	11. Работа native функции IsValidNickName(name[]) полностью зависит от Вашего файла конфигураций;
	12. Обновлены файлы для работы с INI;
	13. Исправлена возможная ошибка в чтении INI файла;
	14. В прошлой версии при отключении хука, плагин завершал работу. Теперь же Вы можете использовать как все хуки одновременно, так и любой один;
	15. Все адреса памяти также находится автоматически, что гарантирует работу на всех современных версиях SA:MP;
	16. Релиз плагина присутствует на 2х языках;
	17. Подправлены шаблоны регулярных выражений. Убрана проверка длины ника. Добавлены новые шаблоны;
	18. Плагин работает и будет работать (надеюсь) на любой версии SA:MP, начиная с 0.3C R5;
	19. Файл конфигураций был переименован с 'ASAN.ini' на 'ASAN_Config.ini';
	20. Файл конфигураций 'ASAN_Config.ini' теперь выглядит так:
		
			Max_Players = 1000
				- Максимальное количество игроков Вашего сервера. НЕ УСТАНАВЛИВАЙТЕ значение меньше, чем установлено в Вашем server.cfg.
				- По умолчанию = 1000

			Language = 1
				- Язык в логах плагина. 0 - Английский, 1 - Русский

			EnableValidNickHOOK = 1
				- Включить хук проверки символов - 1, Отключить хук проверки символов - 0

			RegexTemplate = ([а-яА-ЯёЁ0-9\[\]\_\$\=\(\)\@\.]+)|([a-zA-Z0-9\[\]\_\$\=\(\)\@\.]+)
				- Шаблон регулярного выражения (Regex)! Не включайте в шаблон длину ника!
				- Некоторые шаблоны Вы можете найти в архиве плагина

			MaxAllowdedSpaces = 0
				- Количество замен символов нижнего подчеркивания(_) на символы пробела( )
				- (-1 - нет ограничений(все символы будут заменены), 0 - отключено(По умолчанию))
				- Пример. Если 'MaxAllowdedSpaces = 1' -> 'John_Connor' будет на серверной стороне (В TAB меню) как 'John Connor' после подключения игрока.
				- Другие символы нижнего подчеркивания будут пропущены (John_Connor_Jr станет John Connor_Jr)
				- Будьте осторожны, использую данную функцию с Вашей базой данных!
				- НЕ ИСПОЛЬЗУЙТЕ ПРОБЕЛЫ НА СТОРОНЕ КЛИЕНТА (ОКНО SA-MP)! ТОЛЬКО СЕРВЕРНАЯ СТОРОНА!

			EnableNickLengthHOOK = 1
				- Включить хук длины символов ника - 1. Отключить хук длины символов ника - 0.

			MinNickLength = 3
				- Минимальная длина НикНейма. По умолчанию = 3
				- Значение должно быть > 0!

			MaxNickLength = 20
				- Максимальная длина НикНейма. По умолчанию = 20
				- Не увеличивайте это зачение больше чем 24!

			EnableRepeatedNicksHOOK = 1
				- Включить хук повторных ников - 1, Отключить хук повторных ников - 0
				- Два или более игроков могут использовать одинаковые НикНеймы

			IgnoreRepeatedNicksCase = 1
				- Когда установлено - 1, регистр не проверяется - ник 'приВет' тоже самое, что и 'Привет'. Когда - 0, они считаются разными.
				- Пример. Если 'IgnoreRepeatedNicksCase = 1' Вы можете использовать тот же самый НикНейм (Пример. Алекс и АЛЕКС) в любом регистре одновременно. Если это = 0, только один регистр может быть использован (тот, что подключился первым)
				- Внимание! Эта функция была протестирована только с Русскими и Английскими символами! Никто не знает, как она будет работать с другим языком!

			MaxRepeatedNicks = -1
				- Количество повторных НикНеймов (-1 - нет ограничений, 0 - тоже самое, что и 'EnableRepeatedNicksHOOK = 0')

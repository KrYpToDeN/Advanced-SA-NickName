Версия 4.0
	1. Переименование плагина и всех его составляющих (инклуд, файл настроек) с Vodka_SAMP на Advanced SA NickName (сокращённо ASAN)
	2. Добавлена поддержка стандартов C++11 в makefile
	3. Обновлены инклуды для работы с INI. Код переписан под язык C++ (был под C)
	4. Теперь ник проверяется на валидность с помощью Регулярных выражений (Regex) https://ru.wikipedia.org/wiki/Регулярные_выражения
	5. Проверка ника на валидность с помощью функции 'IsValidNickName(name[])' теперь тоже работает с помощью Regex.
	6. Функция проверки ника 'IsValidNickName(name[])' перенесена в плагин. В инклуде для сервера вынесен native
	7. Исправлен баг в хуке функции 'GetPlayerName(..)' в инклуде. Баг был в том, что функция не возвращала длину ника.
	8. Изменён файл настроек плагина. В него включены следующие настройки:
		
		EnginePlugin
		- Включить плагин - 1, Отключить плагин - 0\n\n\
		
		RegexTemplate
		- Регулярное выражение (Включая регистр символов)\n\n\
		
		AllowSpace
		- Заменить символ нижнего подчёркивания(_) на символ пробела( ) - 1, По умолчанию - 0\n\
		- Пример. Если 'AllowSpace = 1' -> 'John_Connor' будет в табе как 'John Connor' после коннекта. НЕ ИСПОЛЬЗУЙТЕ ПРОБЕЛЫ В ОКНЕ САМПА! ТОЛЬКО СЕРВЕРНАЯ СТОРОНА!\n\n\
		
		MaxSpaces
		- РАБОТАЕТ ТОЛЬКО, ЕСЛИ `AllowSpace` = 1.\n\
		- Количество замен символа (_) на символ (). Тоесть если стоит 1, то система заменит символ только 1 раз, а остальные пропустит!",
	
	9. Некоторые изменения в коде, что добавляет удобства для чтения кода другим программистам.
	
	Для обновления плагина со старых версий, обязательно:
		1. Удалите файл настроек 'Vodka_SAMP.ini', плагин 'Vodka_SAMP.dll' (.so на Linux), инклуд 'Vodka_SAMP.inc'
		2. Положите новый плагин, инклуд в нужные папки.
		3. В Вашем гейм-моде/скриптах измените 'include "Vodka_SAMP"' на 'include "ASAN"'
		4. В сервер.cfg в строке 'plugins' измените 'Vodka_SAMP' на 'ASAN'
